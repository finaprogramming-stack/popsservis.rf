Как рулить бесшовными блоками:

page - номер страницы внутри раздела.
merge - сколько карточек соединить: 2, 3 или 4. Если 0 - блок выключен.
cardCount - сколько визуальных мест должно быть на этой странице: от 1 до 4.
Если поставить 3, четвертая фотка уйдет первой на следующую страницу,
и весь список дальше сдвинется.
photoIds - какие photoId поставить в блок.
При merge 0 photoIds тоже работают как ручная перестановка слотов страницы.
Например было [1,2,3,4], написал [4,2,3,4] - фото 4 встанет
на первый слот, а фото 1 уйдет на старое место фото 4.
title - общий заголовок блока.
description - общее описание блока.
photos - подсказка по текущим фото страницы: слот, photoId, файл, название и описание.

Базовый выключенный шаблон:
[
  {
    "page": 1,
    "merge": 0,
    "cardCount": 4,
    "photoIds": [],
    "title": "",
    "description": "",
    "photos": []
  }
]

Пример:
[
  {
    "page": 1,
    "merge": 3,
    "photoIds": [1, 2, 3],
    "title": "Самосвал и трал: состояние до начала обработки",
    "description": "Общее описание для трех склеенных фото."
  }
]

Если написать photoIds [4, 56, 12], эти фото уйдут в блок.
Фотки, которые раньше стояли в слотах блока, займут старые места перенесенных фото.

Страница считается по визуальным слотам: merge 3 занимает 3 места из 4,
поэтому на этой же странице рядом останется только 1 обычная карточка.
Если при merge 3 поставить cardCount 3, на странице будет только один
бесшовный блок, а следующая фотка уйдет на следующую страницу.

Показ ID на сайте включается тут:
../gallery-settings/show-photo-ids.json

{"showPhotoIds":true} - ID видны в описаниях.
{"showPhotoIds":false} - ID скрыты.
